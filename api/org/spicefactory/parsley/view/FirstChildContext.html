FirstChildContextMethods
 
Packageorg.spicefactory.parsley.view
Classpublic class FirstChildContext
InheritanceFirstChildContext Object

Waits for the first child Context that gets created somewhere in the view hieararchy at or below the specified view instance. The various callbacks allow to either modify the configuration before the Context gets built, or the receive the Context instance once it has either been just created or fully initialized.

bootstrap()method
public function bootstrap(handler:Function):FirstChildContext

The handler to invoke when the bootstrap of the child Context starts. This is a very early callback, the actual Context instance is not available yet. Instead the provided BoostrapConfig instance allows to transparently add configuration options and managed objects to the child Context without the child Context being aware of this. This is primarily useful for developing application shells where some aspects like navigation status might need to be provided to modules or tabs in a transparent way.

The function must accept a parameter of type BootstrapConfig

.
